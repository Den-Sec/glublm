## 1. Executive Summary

**GlubLM Desk Pet** is an interactive pixel-art companion that brings the GlubLM goldfish to life in the browser. A 16x16 pixel goldfish lives in a fishbowl, swims around autonomously, says things on its own, responds when you talk to it, reacts when you click it, and sends push notifications when you're away. It runs the full GlubLM v0.3.1 model (35M params, ONNX 40MB) entirely client-side - zero server, fully offline after first load, installable as a PWA.

The desk pet is the natural evolution of the current `web/` chat demo into something you actually want to keep open.

---

## 2. Vision

> A goldfish you adopt in your browser. It swims, it talks to itself, it forgets everything. You can talk to it and it'll respond - then forget. You can poke it and it'll wiggle. Leave it alone long enough and it'll fall asleep. Close the tab and it'll tap on the glass asking where you went.

**Primary platform**: Browser (desktop + mobile).
**Secondary** (future, not in scope): Raspberry Pi with e-ink/LCD, ESP32 with LED matrix.

## 3. Core Experience

### 3.1 The Bowl

The entire UI is a fishbowl. No chrome, no menus, no sidebar. The bowl IS the application.

- Round/oval bowl rendered on canvas
- Water with subtle gradient (lighter at top, deeper blue at bottom)
- Gravel at the bottom (static pixel pattern)
- 1-2 small plants (static sprites)
- Ambient bubbles rising slowly (canvas particle system)
- Subtle water surface ripple at the top
- Glass reflection overlay (very subtle)

### 3.2 The Fish

A 16x16 pixel-art goldfish that lives in the bowl.

**Movement:**
- Swims freely within bowl bounds
- Random target positions, smooth interpolation (lerp)
- Pauses at target, picks new target
- Flips horizontally based on direction
- Tendency to stay mid-bowl (gravity-like pull)
- Occasionally bumps the glass edge

**Animation states (12):**

| State | Frames | FPS | Trigger | Duration |
|-------|--------|-----|---------|----------|
| `idle_swim` | 4 | 6 | Default | Loop |
| `talk` | 4 | 8 | User message / idle phrase | Variable |
| `happy` | 4 | 8 | Positive interaction | 2-3s |
| `sad` | 3 | 4 | Sad response context | 2-3s |
| `sleep` | 4 | 2 | Inactivity >5 min | Until interaction |
| `eat` | 4 | 6 | Random / user click on food | 2s |
| `bump_glass` | 4 | 6 | Random / reaches bowl edge | 1.5s |
| `forget` | 3 | 4 | Post-chat / random | 2s |
| `excited` | 4 | 10 | Double-click / surprise | 1.5s |
| `wiggle` | 3 | 8 | Single click on fish | 1s |
| `bubble_blow` | 5 | 6 | Random | 2s |
| `turn_around` | 4 | 6 | Direction change | 0.5s |

**Sprite sheet specification:**
- Single PNG, grid layout
- Each row = one animation state
- Each column = one frame
- Cell size: 16x16 pixels
- Rendered at 8x scale (128x128 on screen) with nearest-neighbor interpolation
- Estimated sheet size: ~80x192 pixels (12 rows x 5 max cols)
- Metadata as JS object mapping state -> { row, frames, fps, loop }

### 3.3 Speech Bubbles

Pixel-art style speech bubble rendered in canvas:
- Appears above/beside the fish
- Pixel-art border (1px black, white fill)
- Text rendered with a small pixel-like font (or canvas fillText at appropriate size)
- Auto-dismiss after 5-8 seconds (configurable per phrase length)
- Fade-in/fade-out animation
- Max width constrained to prevent overflow

### 3.4 Interactions

**Text input:**
- Minimal textbox at bottom of the bowl
- Semi-transparent, doesn't dominate the scene
- Mobile: appears on tap (auto-focus), hides on blur
- Desktop: always visible but subtle
- Submit via Enter or send button

**Click/touch on fish:**
- Single tap: WIGGLING state
- Double tap: EXCITED state
- Long press (>500ms): HAPPY state ("petting")
- Drag: fish follows finger briefly, then swims away

**Click on bowl (not fish):**
- Tap: subtle ripple effect on water surface
- No fish reaction (it's just the glass)

### 3.5 Idle Behavior

The fish has a life of its own. When the user isn't interacting:

**Phrase pool (500+ pre-generated):**
- Stored in `data/idle-phrases.json`
- Generated with Claude to match GlubLM's tone (conversational, forgetful, goldfish-naive)
- NOT generated by the ONNX model (those are for live chat only)

**Categories:**

| Category | Time/Context | Example |
|----------|-------------|---------|
| `morning` | 6:00-12:00 | "oh! the light came back. i love when that happens" |
| `afternoon` | 12:00-18:00 | "i've been swimming in circles. or maybe a square. hard to tell" |
| `evening` | 18:00-22:00 | "the light is getting softer. that's nice. what was i saying" |
| `night` | 22:00-6:00 | "everything is quiet. i like quiet. wait what's quiet" |
| `bored` | After 2-5min silence | "hello? is someone there? i forgot" |
| `hungry` | Random + eating anim | "i could go for some flakes right about now" |
| `post_chat` | After conversation | "that was nice. what were we talking about?" |
| `long_silence` | After 10+ min | "i've been alone for... how long? doesn't matter. water's nice" |
| `existential` | Random | "do you think the bowl is the whole world? asking for a fish" |
| `meta` | Random | "sometimes i see a face in the glass. big face. weird" |
| `cheerful` | Random | "blub blub blub. that's my favorite song" |
| `forgetful` | Random | "i was going to say something important. wait. was i?" |

**Selection algorithm:**
1. Filter by time-of-day category (always include time-neutral categories)
2. Weight post_chat/long_silence based on context (recent chat? long idle?)
3. Weighted random within filtered pool
4. Exclude last 20 shown phrases (no repeats)
5. Interval: 30-120 seconds (randomized, Poisson-like distribution)

### 3.6 Chat with the Fish

When the user types a message:

1. Message appears briefly as user speech bubble (bottom of bowl)
2. Fish enters TALKING state (mouth animation)
3. ONNX model runs inference (existing SimpleBPE + sampling logic)
4. Response appears as fish speech bubble
5. Fish transitions to contextual state (HAPPY/FORGETTING/etc.)
6. After 5-8 seconds, speech bubble fades
7. Fish returns to IDLE

**No chat log.** The conversation disappears - matching the "forgets everything" identity. The bowl shows only the current moment.

## 4. Technical Architecture

### 4.1 Rendering Pipeline

```
Canvas 2D (single <canvas> element, full viewport)
│
├── Layer 0: Bowl background (gradient + gravel + plants)
├── Layer 1: Water effect (subtle wave at surface)
├── Layer 2: Ambient bubbles (particle system, 5-8 bubbles)
├── Layer 3: Fish sprite (16x16 scaled to 128x128)
├── Layer 4: Speech bubble + text
├── Layer 5: UI overlay (input field via DOM, not canvas)
└── Layer 6: Glass reflection (subtle)
```

**Frame timing:**
- requestAnimationFrame for the render loop (60fps capable)
- Fish position updates: every frame (smooth movement)
- Sprite frame changes: throttled to state-specific FPS (4-10)
- Bubble particles: every frame
- Background: re-render only on resize

### 4.2 State Machine

```
                    ┌─────────────────────────────────────────────┐
                    │                                             │
                    ▼                                             │
              ┌──────────┐    click    ┌───────────┐              │
         ┌───>│   IDLE   │───────────>│  WIGGLING  │──────────────┤
         │    │ (swim)   │            └───────────┘              │
         │    └────┬─────┘                                       │
         │         │                                             │
         │    ┌────┼────┬────────┬──────────┬──────────┐         │
         │    │    │    │        │          │          │         │
         │    │  msg  timer   dbl-click  timer>5m   timer       │
         │    │    │    │        │          │          │         │
         │    │    ▼    ▼        ▼          ▼          ▼         │
         │    │ TALKING BUMP  EXCITED   SLEEPING  BUBBLE_BLOW   │
         │    │    │   GLASS     │          │          │         │
         │    │    │    │        │          │          │         │
         │    │    ▼    │        │       click/msg     │         │
         │    │ HAPPY/  │        │          │          │         │
         │    │ SAD/    │        │          │          │         │
         │    │ FORGET  │        │          │          │         │
         │    │    │    │        │          │          │         │
         │    └────┴────┴────────┴──────────┴──────────┘         │
         │         │                                             │
         └─────────┴─────────────────────────────────────────────┘
                 (all states return to IDLE after duration)
```

**State priorities (higher = can interrupt lower):**
1. User interactions (click, message) - priority 3
2. Idle phrases (auto-talk) - priority 2
3. Random animations (bump, bubble, turn) - priority 1
4. IDLE/SLEEPING - priority 0

### 4.4 Canvas Rendering Details

**Scaling strategy:**
- Base resolution: 320x400 (logical pixels)
- Canvas scales to fit viewport while maintaining aspect ratio
- All pixel art rendered at integer multiples (no sub-pixel)
- CSS `image-rendering: pixelated` on canvas element
- DPI-aware: `devicePixelRatio` for sharp rendering on retina

**Bowl geometry (in logical pixels):**
- Bowl center: (160, 220)
- Bowl radius X: 120, radius Y: 140 (slightly oval)
- Water surface: y=100 (wave animation)
- Gravel line: y=340
- Fish swim zone: y=120 to y=320, x=60 to x=260

### 4.5 Inference Integration

Reuse the existing `glub.js` logic, refactored into ES modules:

```javascript
// inference/model.js
export class OnnxModel {
  constructor() { this.session = null; this.tokenizer = null; }
  async load(modelUrl, tokenizerUrl, onProgress) { ... }
  async generate(prompt, { maxNewTokens, temperature, topK }) { ... }
  get isReady() { ... }
}

// inference/tokenizer.js
export class SimpleBPE {
  constructor(json) { ... }   // existing SimpleBPE from glub.js
  encode(text) { ... }
  decode(ids) { ... }
}
```

**Loading UX:**
- Show "adopting goldfish..." with progress bar during model download
- Model cached by Service Worker after first load
- Subsequent visits: instant load from cache
- If model not yet loaded: idle phrases still work (no inference needed)
- Input disabled until model ready

```json
{

  "phrases": [
    {

      "category": "morning",
      "weight": 1.0
    },
    {
      "text": "that was nice. what were we talking about?",
      "category": "post_chat",
      "weight": 1.0
    }

```

**Category → time mapping (in IdleScheduler):**

| Category | Active hours | Context condition |
|----------|-------------|-------------------|
| morning | 06:00-12:00 | - |
| afternoon | 12:00-18:00 | - |
| evening | 18:00-22:00 | - |
| night | 22:00-06:00 | - |
| bored | any | silence > 2min |
| hungry | any | random 10% chance |
| post_chat | any | last chat < 2min ago |
| long_silence | any | silence > 10min |
| existential | any | random 5% chance |

### 4.7 PWA Architecture

**manifest.json:**
```json
{
  "name": "GlubLM Desk Pet",
  "short_name": "Glub",
  "description": "A goldfish companion that already forgot this description",
  "start_url": ".",
  "display": "standalone",
  "background_color": "#e5f4ff",
  "theme_color": "#ff8b3d",
  "icons": [
    { "src": "assets/icons/icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "assets/icons/icon-512.png", "sizes": "512x512", "type": "image/png" }
  ]
}
```

**Service Worker caching strategy:**
- **Install**: pre-cache HTML, CSS, JS, sprites, idle-phrases.json, icons
- **Activate**: clean old caches
- **Fetch**: cache-first for all static assets
- **Model/tokenizer**: cache on first fetch (too large for pre-cache, use streaming cache)
- **ONNX Runtime**: cache from CDN on first load

**Notification approach (no server):**
- Use `Notification` API + `ServiceWorkerRegistration.showNotification()`
- Trigger via `setInterval` in Service Worker (while alive)
- `visibilitychange` event: when tab goes to background, schedule notifications
- Phrases selected from the idle pool (subset tagged `notification`)
- Frequency: configurable (default: 3-4 per day)
- Limitation: notifications stop when SW is killed (browser closes completely). Documented as known limitation. Future: Cloudflare Worker push relay.

## 6. Performance Targets

| Metric | Target |
|--------|--------|
| First paint (cached) | < 500ms |
| Model load (cached) | < 2s |
| Model load (network, 40MB) | Progress bar, no timeout |
| Idle CPU usage | < 5% (canvas + timer) |
| Memory footprint | < 80MB (model + runtime + canvas) |
| Sprite animation | Consistent 8 FPS frame changes |
| Canvas render | 60 FPS smooth |
| Inference latency | < 3s per response (WASM) |
| Service Worker install | < 1s (excluding model) |
| Total asset size (excl. model) | < 500KB |

## 7. Non-Goals

- Multi-turn memory (the fish forgets - that's the point)
- Chat history / conversation log
- Cloud sync / accounts / login
- Multiplayer / shared bowls
- Fish customization / skins (possible future feature)
- Sound effects (Phase 2+ consideration, not MVP)
- Hardware port (RPi/ESP32) - evaluated separately after browser version ships
- Backend / server of any kind
- Build tools / bundlers / transpilers
- Framework (React/Vue/etc.) - vanilla JS only

## 8. Success Criteria

### 8.1 Must-haves (MVP)

- [ ] Fish swims in a pixel-art bowl with smooth movement
- [ ] All 12 animation states working with sprite transitions
- [ ] Click/tap on fish triggers reaction
- [ ] User can type and get a response from the ONNX model
- [ ] Fish says idle phrases on its own (time-aware)
- [ ] Speech bubbles appear and auto-dismiss
- [ ] PWA installable (manifest + service worker)
- [ ] Works fully offline after first load
- [ ] Mobile-responsive layout
- [ ] Model cached by service worker (no re-download)

### 8.2 Nice-to-haves

- [ ] Push notifications when tab is in background
- [ ] Custom fish name (localStorage)
- [ ] Settings panel for notification frequency
- [ ] PWA install prompt ("adopt this goldfish")
- [ ] Loading progress bar for model download
- [ ] Ambient sound toggle

### 8.3 Stretch

- [ ] Drag-to-interact (fish follows finger)
- [ ] Seasonal themes (bowl decorations based on date)
- [ ] Fish "mood" system (accumulates over interactions)
- [ ] Share button (screenshot of bowl)

## 9. Open Questions

1. **Sprite art**: who draws the pixel art? Dennis, commissioned artist, or AI-generated? Engine should work with any 16x16 sheet.
2. **Idle phrase generation**: use Claude to batch-generate 500+ phrases, or curate manually? Recommend Claude with manual review.
3. **Notification frequency default**: 3/day feels right but needs user testing.
4. **Bowl shape**: perfect circle or slight oval? Oval feels more natural for a desktop fishbowl.
5. **Deployment**: separate GitHub Pages path (`/desk-pet/`) or replace current demo at root?
6. **Model.onnx**: copy into `desk-pet/` or reference from `web/`? Copy is cleaner for independent deploy.
