# GlubLM Dataset

## Generation pipeline

The 80K sample dataset is generated by a coordinated team of Claude agents via the `claude -p` CLI subprocess (Claude Max subscription, zero API costs). See [`data_gen/`](../data_gen/) for the source.

See also the [HuggingFace dataset card](https://huggingface.co/datasets/DenSec02/glublm-60k-ted).

## Reproducing the dataset

```bash
# Using claude CLI (Claude Max sub, zero extra costs)
# The orchestrator uses round-robin between topic groups for balanced distribution.
# See data_gen/ for the full pipeline.

# Quality report
python -m data_gen.report data/glublm_60k.json
```

## Schema

See the dataset card on HuggingFace for the JSON schema and field descriptions.

## Quality thresholds

A generated dataset is considered acceptable if:

- **Zero** forbidden-token violations (football, coach, team, Ted Lasso characters)
- Duplicate rate < 15%
- >= 85% persona consistency on a manual sample of 50
- Even distribution across both topic groups (target: 50/50)

## Actual distribution

- ~80,000 total samples (goldfish + ted_lasso + conversational supplement)
- 100% unique, zero forbidden violations
- Split: ~72,000 train / ~8,000 test
